# プレゼン用マスタ資料（外部要件）

AI スライド生成ツールに入力することを想定し、以下の箇条書きはそのままコピペできる形式で整理しています。各セクションは 1〜2 枚のスライドに展開できるよう、見出し→箇条書きの構成にしています。

---
## 1. 目的と背景
- 組織タスクフローのボトルネック可視化
- 役職別の負荷集中ポイントを定量評価
- 組織改編やヒアリング結果を迅速に検証するための社内 PoC

---
## 2. 想定ユーザー
- 組織構造分析チーム
- 経営企画／HR などの組織設計担当
- 研究開発チーム（インタビュー結果をシミュレーションに落とし込み）

---
## 3. 出力物
- `RESULT.md`：ノード全件の指標（タスク数 / SimLoad / DegreeCentrality / Score）
- `REPORT.md`：役職別トップスコアの抜粋
- 必要に応じて CSV/画像へ変換（外部要望に応える拡張余地あり）

---
## 4. 実行環境
- OS：macOS または Linux
- Python：3.11 以上推奨
- 必須ライブラリ：`networkx`
- 任意：`matplotlib`（組織図や図表を出力する場合）
- 仮想環境：
  1. `python3 -m venv .venv`
  2. `source .venv/bin/activate`
  3. `pip install -r requirements.txt`

---
## 5. データ・モデル前提
- 階層：CXO → Director → SeniorManager → Manager → Player（5層ツリー）
- 親1人あたりの部下数（例）：Director 15〜20、SeniorManager 10〜15、Manager 10〜15、Player 5〜10
- タスク生成：役職ごとの発生確率 × Poisson(λ=2)、乱数シードで再現性を確保
- SimLoad：受信/送信/報告ごとに負荷加算、Score＝0.7×正規化SimLoad＋0.3×DegreeCentrality

---
## 6. 外部連携・レポート要件
- 入力：組織グラフはコード内で自動生成（外部ファイル取込みは現状なし）
- 出力：Markdown（AIスライドやBIへの転用が容易）、将来的に CSV/JSON API を検討
- レビュー用途：`REPORT.md` で要点を抜粋、`RESULT.md` で全明細を保持

---
## 7. 運用・保守観点
- 依存ライブラリ未導入時のエラーメッセージを明確化
- Markdown の行数増大に備え、閲覧は VS Code 等のプレビュー推奨
- 役職ごとの平均値が欠落しないよう、Node Details から自動補完する処理を実装

---
## 8. 今後の外部要求想定
- CSV/JSON など外部システム連携フォーマットの追加
- シミュレーション条件を YAML/JSON 設定に切り出し、非エンジニアでも編集可能に
- BI ツール（Looker, Power BI 等）への自動連係、ダッシュボード化

---
このまま AI に投入すれば、「どのスライドで何を説明するか」が生成しやすくなります。必要に応じて各項目を 1 枚のスライドに展開し、図や数値を補足してください。
